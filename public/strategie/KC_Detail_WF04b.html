WF-04b — KC_CallManagement
Gestion des non-réponses aux appels Sofia — 3 tentatives automatiques
Construit
Workflow 04b
KC_CallManagement
Quand Sofia ne parvient pas à joindre un contact, ce workflow relance automatiquement jusqu'à 3 fois sur 27 heures, puis envoie un SMS de dernier recours avant de clôturer le dossier.
✅ Statut : Construit 🔁 3 tentatives max ⏱️ Durée : 27h max
⚡ Déclencheur
Type de trigger
Call-End Workflow de Sofia (sur non-réponse)
Ce workflow n'a pas de trigger direct — il est appelé par Sofia (KC_Outbound_Prospector) comme "Call-End Workflow" uniquement lorsque l'appel n'a reçu aucune réponse.
⚠️ À faire : dans les paramètres de Sofia → Post-Call → Call-End Workflow → sélectionner "KC_CallManagement"
🔁 Logique de retry — 3 tentatives
1Tentative

Premier appel Sofia → Pas de réponse

Tag KC_reactiv-no-answer ajouté. WF-04b démarre. Attente de 3 heures avant la 2e tentative.

3hAttente

Pause de 3 heures

Le workflow attend 3h pour respecter le temps du prospect. Après l'attente, Sofia rappelle une 2e fois.

2Tentative

Deuxième appel Sofia → Pas de réponse

Toujours sans réponse. Attente de 24 heures avant la 3e et dernière tentative.

24hAttente

Pause de 24 heures

Délai plus long pour le lendemain — le prospect est peut-être plus disponible.

3Tentative

Troisième appel Sofia → Pas de réponse

Tag KC_reactiv-retry-final ajouté pour signaler que les 3 tentatives sont épuisées.

📵 Après 3 tentatives sans réponse

SMS de dernier recours envoyé au client : "Bonjour, on a essayé de vous joindre par téléphone. Si vous êtes toujours intéressé par nos services, n'hésitez pas à nous rappeler au (514) 591-6202." → Pipeline : PERDU

🏷️ Tags utilisés
KC_reactiv-no-answer KC_reactiv-retry-final KC_perdu
🛡️ Protection anti-boucle infinie

Pourquoi 2 tags de comptage ?

KC_reactiv-no-answer = tentative 1 épuisée. KC_reactiv-retry-final = tentative 3 épuisée. Le workflow vérifie ces tags à chaque étape pour ne jamais dépasser 3 appels.
