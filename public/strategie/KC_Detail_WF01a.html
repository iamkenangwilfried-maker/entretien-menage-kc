Workflow 01a
KC_Form_Inbound
Capture automatiquement chaque soumission du formulaire sur le site web KC et déclenche la séquence de bienvenue complète.
✅ Statut : Live 📥 Source : Formulaire site web 🎯 Pipeline : Nouveau Lead
⚡ Déclencheur
Type de trigger
Form Submitted
Se déclenche dès qu'un visiteur soumet le formulaire de demande de soumission sur le site web KC (widget live call / formulaire de contact).
📋 Étapes du workflow
1

Créer / Mettre à jour le contact

GHL crée un nouveau contact avec les informations du formulaire (Prénom, Nom, Email, Téléphone). Si le contact existe déjà, il est mis à jour.

2

Ajouter les tags source

Ajout automatique de KC_src-widget (origine : widget/formulaire) et KC_live-widget-lead (statut : lead actif).

3

Créer une opportunité dans le pipeline

Une nouvelle opportunité est créée dans le pipeline KC_Entretien à l'étape Nouveau Lead. Le lead est maintenant visible dans le CRM.

4

SMS de confirmation au client

Le client reçoit immédiatement un SMS de confirmation en français : accusé de réception, délai de rappel, numéro KC.

5

Email de confirmation au client

Email de bienvenue brandé KC avec récapitulatif de la demande et prochaines étapes.

6

Email de notification interne

Alerte immédiate envoyée à kcglobalservice.inc@gmail.com avec toutes les informations du nouveau lead pour callback rapide.

🏷️ Tags utilisés
KC_src-widget KC_live-widget-lead
Ces tags permettent de filtrer tous les leads provenant du formulaire web et de déclencher d'autres automatisations en aval.
🔗 Connexions système

Pipeline alimenté

KC_Entretien → étape Nouveau Lead

Workflows déclenchés en aval

WF-Devis se déclenche automatiquement quand l'équipe fait avancer l'opportunité à "Devis Envoyé".
