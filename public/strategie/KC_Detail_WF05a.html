Workflow 05a
KC_ReviewRequest_Sequence
Dès qu'un service est complété, ce workflow envoie jusqu'à 5 SMS pour inviter le client à noter KC. Les 4-5 étoiles vont directement sur Google, les avis négatifs sont capturés en privé.
✅ Statut : Live ⭐ Google Reviews 5 SMS max
🌐 Page d'avis KC — À montrer à la cliente
https://entretien-menage-kc.vercel.app/demande-avis Ouvrir →

⭐⭐⭐⭐⭐ Avis 4-5 étoiles

Le client clique sur 4 ou 5 étoiles → redirigé directement vers la page Google Review de KC pour laisser un avis public.

⭐⭐⭐ Avis 1-3 étoiles

Formulaire de feedback privé → envoyé à l'équipe KC via webhook → WF-05b gère l'alerte interne. L'avis négatif ne va PAS sur Google.

⚡ Déclencheur
Type de trigger
Opportunity Stage Changed → "Service Complété"
Dès que l'équipe KC marque un service comme complété dans le pipeline, la séquence de demande d'avis démarre automatiquement.
📱 Séquence de 5 SMS
Immédiat

SMS 1 — Premier contact

SMS chaleureux avec le Trigger Link vers /demande-avis. Si le client clique → stop.

J+2

SMS 2 — Rappel doux

Deuxième invitation. Vérification : si KC_review-link-clicked → skip ce SMS.

J+5

SMS 3 — Suivi

Troisième invitation avec une formulation différente. Stop si déjà cliqué.

J+10

SMS 4 — Rappel final

Dernier SMS avant clôture de la séquence. Formulation urgente mais respectueuse.

J+14

SMS 5 — Dernier recours

SMS de clôture. Après ce message la séquence se termine automatiquement.

🛑 Stop automatique : à chaque SMS, le workflow vérifie si le tag KC_review-link-clicked est présent. Si oui, la séquence s'arrête immédiatement — aucun SMS supplémentaire.
🔀 Architecture de gating (filtrage des avis)
📱 Client reçoit le Trigger Link SMS → clique
→
⭐⭐⭐⭐⭐ 4-5 étoiles → Google Review public
⭐⭐⭐ 1-3 étoiles → Formulaire privé → WF-05b → alerte interne
