WF-04b — KC_CallManagement

Gestion des no-answer Sofia — 2 retentatives automatiques puis SMS fallback

⚡ TRIGGER : CALL-END WORKFLOW — Post-Call de Sofia Outbound
✅ Où configurer le trigger dans GHL AI Agents → Voice AI → Sofia Outbound → Post-Call → Call-End Workflow → KC_CallManagement
Ce workflow se déclenche après chaque fin d'appel Sofia (qu'il soit répondu ou non).
Les 4 premiers IF/ELSE filtrent les contacts déjà traités — seuls les vrais no-answer continuent.
⚠️ 1 nouveau tag à créer dans GHL avant de construire ce workflow KC_reactiv-retry-final — marque la 2ème tentative de retry (empêche une boucle infinie).

Notes importantes
Pas de boucle infinie : Le tag KC_reactiv-retry-final agit comme disjoncteur — au 3ème no-answer, WF-04b détecte le tag au Node 3 et envoie le SMS fallback au lieu de rappeler.

Nœuds IF/ELSE sur tags : Contrairement aux conditions sur les appels Voice AI, les conditions sur les tags sont pleinement supportées dans GHL — pas besoin de Custom Webhook.

Après construction : Aller dans Sofia Outbound → Post-Call → Call-End Workflow → sélectionner KC_CallManagement → Save.
