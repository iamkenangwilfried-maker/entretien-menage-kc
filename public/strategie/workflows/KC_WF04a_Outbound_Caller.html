⚙️ Actions configurées dans Sofia (pendant l'appel)

1
Update contact field → KC_Type_de_service
Quand le prospect mentionne le type de service : commercial, résidentiel, industriel, médical, post-construction, déménagement.
2
Update contact field → KC_Ville
Quand le prospect mentionne sa ville ou son secteur géographique.
3
Update contact field → KC_Surface_m2
Quand le prospect mentionne la superficie de son espace.
4
Update contact field → KC_Frequence
Quand le prospect mentionne la fréquence souhaitée : hebdomadaire, bihebdomadaire, mensuel, ponctuel.
Notes importantes
Consent Tool obligatoire : GHL bloque les appels sortants Voice AI si le texte de consentement n'est pas rempli et validé dans AI Agents → Voice AI → Outbound Calling Consent Tool.
Créneaux d'appel : Configurer Sofia pour appeler uniquement entre 9h et 18h lundi-vendredi — règles CRTC québécoises.
Pas de IF/ELSE dans le workflow : Le routing est entièrement géré par le Post-Call de Sofia — plus simple et plus fiable que les conditions GHL sur les appels Voice AI.
WF-01a réutilisé : Dès que Sofia qualifie un contact, WF-01a prend le relais — SMS confirmation + email + notif interne + pipeline Nouveau Lead. Aucune duplication.
WF-04b : Doit être créé et publié avant de tester WF-04a.
